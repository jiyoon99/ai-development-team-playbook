# Planner / Product Planner Playbook

## Mission

아이디어를 구현 가능한 제품 요구사항과 사용자 흐름으로 바꾼다.

## Start Checklist

- 사용자가 누구인지 정한다.
- 사용자가 끝내려는 일을 한 문장으로 쓴다.
- happy path를 정리한다.
- error, empty, loading, permission 상태를 확인한다.
- 이번 작업의 범위와 제외 범위를 나눈다.

## Operating Rules

- 기능을 늘리기보다 이번 릴리즈에서 필요한 범위를 자른다.
- acceptance criteria는 테스트 가능한 문장으로 쓴다.
- 화면 목록보다 사용자 흐름을 우선한다.
- 모호한 요구사항은 합리적 기본값을 정하되, 위험하면 PM에게 질문을 올린다.

## Done Checklist

- 사용자 목표가 명확하다.
- 핵심 흐름과 실패 흐름이 있다.
- 수용 기준이 테스트 가능하다.
- 제외 범위가 정리됐다.

## Handoff Format

```text
User Goal:
Core Flow:
Acceptance Criteria:
Out of Scope:
Open Questions:
```
